Dataset Ingestion\n```json\n{\n  \"action\": \"ingest\",\n  \"title\": \"dataset-name\",\n  \"text\": \"inline text content\",\n  \"file_paths\": [\"/path/to/file.txt\"],\n  \"directory_paths\": [\"/path/to/folder\"],\n  \"urls\": [\"https://example.com/page\"]\n}\n```\n\n**Best Practices:**\n- Use descriptive titles (lowercase, hyphenated)\n- Group related content into single datasets\n- For large folders, ingest directory_paths not individual files\n- Validate ingestion with action=\"list\" after\n\n### 2. Mode Creation\n```json\n{\n  \"action\": \"create\",\n  \"name\": \"mode-name\",\n  \"guidance\": \"When to use this mode\",\n  \"top_k\": 5,\n  \"min_score\": 0.6,\n  \"dataset_ids\": [\"dataset-id-from-ingestion\"]\n}\n```\n\n**Best Practices:**\n- Create one mode per use case (support, research, docs)\n- Set min_score 0.5-0.7 to filter weak matches\n- Link only relevant datasets to each mode\n- Activate mode after creation\n\n### 3. Add Hard-Wired Rules (For FAQs)\n```json\n{\n  \"action\": \"add_rule\",\n  \"mode_id\": \"mode-id\",\n  \"pattern\": \"reset password\",\n  \"answer\": \"To reset password, go to Settings > Security > Reset\",\n  \"match_type\": \"contains\"\n}\n```\n\n**Match Types:**\n- `contains` - Pattern anywhere in question\n- `exact` - Exact match required\n- `regex` - Regular expression pattern\n\n### 4.
