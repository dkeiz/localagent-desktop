Dataset Ingestion\n```json\n{\n  \"action\": \"ingest\",\n  \"title\": \"dataset-name\",\n  \"text\": \"inline text content\",\n  \"file_paths\": [\"/path/to/file.txt\"],\n  \"directory_paths\": [\"/path/to/folder\"],\n  \"urls\": [\"https://example.com/page\"]\n}\n```\n\n**Best Practices:**\n- Use descriptive titles (lowercase, hyphenated)\n- Group related content into single datasets\n- For large folders, ingest directory_paths not individual files\n- Validate ingestion with action=\"list\" after\n\n### 2. Mode Creation\n```json\n{\n  \"action\": \"create\",\n  \"name\": \"mode-name\",\n  \"guidance\": \"When to use this mode\",\n  \"top_k\": 5,\n  \"min_score\": 0.6,\n  \"dataset_ids\": [\"dataset-id-from-ingestion\"]\n}\n```\n\n**Best Practices:**\n- Create one mode per use case (support, research, docs)\n- Set min_score 0.5-0.7 to filter weak matches\n- Link only relevant datasets to each mode\n- Activate mode after creation\n\n### 3. Add Hard-Wired Rules (For FAQs)\n```json\n{\n  \"action\": \"add_rule\",\n  \"mode_id\": \"mode-id\",\n  \"pattern\": \"reset password\",\n  \"answer\": \"To reset password, go to Settings > Security > Reset\",\n  \"match_type\": \"contains\"\n}\n```\n\n**Match Types:**\n- `contains` - Pattern anywhere in question\n- `exact` - Exact match required\n- `regex` - Regular expression pattern\n\n### 4. Activate Mode\n```json\n{\n  \"action\": \"activate\",\n  \"mode_id\": \"mode-id\"\n}\n```\n\n### 5. Query Dataset\n```json\n{\n  \"query\": \"user question here\",\n  \"mode_id\": \"optional-mode-override\",\n  \"top_k\": 3\n}\n```\n\n## Common Mistakes to Avoid\n\n| Mistake | Fix |\n|---------|-----|\n| Querying without active mode | Always activate mode first or specify mode_id |\n| Ingesting without checking result | Use action=\"inspect\" to verify chunks |\n| Too low min_score | Set 0.5+ to avoid irrelevant matches |\n| Mixing unrelated content | Keep datasets topic-specific |\n| Forgetting to list datasets | Use action=\"list\" to get dataset_ids |\n\n## Quick Reference Commands\n\n```\n# Check status\n{\"action\":\"status\"}\n\n# List datasets\n{\"action\":\"list\",\"type\":\"datasets\"}\n\n# List modes\n{\"action\":\"list\",\"type\":\"modes\"}\n\n# Inspect dataset\n{\"action\":\"inspect\",\"dataset_id\":\"ds-xxx\"}\n```\n\n## Example: TechSupport Dataset Setup\n\n1.
